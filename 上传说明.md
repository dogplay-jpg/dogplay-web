# 快速上传代码到 GitHub

由于 Git 认证问题，请使用以下方法上传代码：

## 方法 A：使用 GitHub Desktop（最简单）

1. 下载安装 GitHub Desktop：https://desktop.github.com/
2. 用你的 GitHub 账号登录 (dogplay-jpg)
3. 点击 File → Add Local Repository
4. 选择 E:\dpy1 文件夹
5. 点击 Publish repository
6. 选择 dogplay-jpg/dogplay-web
7. 点击 Publish

## 方法 B：GitHub 网页上传

### 步骤 1：删除现有仓库（如果需要）

1. 访问 https://github.com/dogplay-jpg/dogplay-web/settings
2. 滚动到最底部 "Danger Zone"
3. 点击 "Delete this repository"
4. 输入仓库名称确认删除

### 步骤 2：创建新仓库并上传

1. 访问 https://github.com/new
2. 仓库名：dogplay-web
3. 选择 "Add a README file"
4. 点击 "Create repository"

### 步骤 3：上传代码

#### 使用 GitHub 上传工具：

1. 在仓库页面点击 "uploading an existing file"
2. 将 E:\dpy1 文件夹中的所有文件拖进去
3. 或使用 GitHub CLI

#### 使用 GitHub CLI（推荐）：

```bash
# 安装 GitHub CLI
# 下载：https://cli.github.com/

# 登录
gh auth login

# 选择 GitHub.com
# 选择 HTTPS
# 选择 Yes (上传 SSH key)
# 浏览器完成授权

# 进入项目目录
cd E:/dpy1

# 创建仓库并推送
gh repo create dogplay-web --public --source=. --remote=origin --push
```

## 方法 C：更改 Git 配置（推荐）

在项目目录运行：

```bash
cd E:/dpy1

# 移除旧的远程仓库
git remote remove origin

# 添加新的远程仓库（使用你的账号）
git remote add origin https://dogplay-jpg@github.com/dogplay-jpg/dogplay-web.git

# 推送代码
git push -u origin main
```

如果提示输入密码，需要使用 Personal Access Token：
1. 访问 https://github.com/settings/tokens
2. 生成新 token，勾选 repo 权限
3. 用 token 作为密码

## 上传成功后

访问 https://github.com/dogplay-jpg/dogplay-web/actions 查看 GitHub Actions 自动部署状态！
